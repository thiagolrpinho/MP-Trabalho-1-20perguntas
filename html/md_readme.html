Game of 20 Questions

The game consists of a series of Yes or No questions until it get on an answer. If the answer and what the user thought at the start of the game are equal, the program wins. If they differ, the game will ask the player for the right answer and which question answering yes would lead to this right answer. The game will get more questions and right answer after more plays. It's possible to save a game, load an anterior game, to start a new game or to edit an already loaded one.

How to play

First clone this repository.

Prerequisites

You need g++ installed which is native from ubuntu.

Installing

After extracting it, enter on the folder using your terminal and write

1 make

Play it

After that, write on your console:

1 ./play_game

Running the tests

On the folder you installed your Game of 20 Questions, write in the terminal: To make a program to run all the tests:

1 make all_tester

If you want to see the tests before running it, write:

1 ./all_tester --list-tests

Or just run it using

1 ./all_tester

Warning The all_tester is not fully automatized because I'm new to catch library of tests and don't know how to automatize input and output on terminal tests. This happens only with tests_game_interface.cpp

Running other tests

To run other tests just go to the main folder of the project: To test only the Binary Tree used on the game.

1 make btree_tester

Beside the previous tests, if you want to test too the game statements of the game:

1 make game_statement_tester

If you want to test too the game engine:

1 make game_engine_tester

Gcov Test

If you have gcov and gcovr installed, you can generate the test coverture of the binary tree by typing:

1 make gcov_btree

And then:

1 ./gcov_btree

After this you can type:

1 make gcovr_html

You can additionally check game statements and game engine by changing gcov_btree for gcov_statement or gcov_engine respectively on the previous list of instruction. You can also just genereate for all tests by doing:

1 make gcov_tester

And then:

1 ./gcov_tester

After this you can type:

1 make gcovr_html

To generate a graphical representantion of how much the code is abranged by the tests.
